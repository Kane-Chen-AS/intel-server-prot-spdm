mctp_spdm.split_multi_random_1(payload_size, maxsize=64)

Split a payload size as multiple segments

Follow MCTP Base Spec. – Incorrect transmission unit An implementation may terminate message assembly if it receives a “middle” packet (SOM = 0b and EOM = 0b) where the MCTP packet payload size does not match the MCTP packet payload size for the start packet (SOM = 1b and EOM bit = 0b). This is considered an error condition. {SOM = 1, EOM = 0}: First packet {SOM = 0, EOM = 0}: Middle packet (equal to first packet size) {SOM = 0, EOM = 0}: Middle packet (equal to first packet size) {SOM = 0, EOM = 1}: Last packet

No more than 4 pkts in a MCTP message

Parameters:

payload_size – payload size in bytes

Returns:

lst of random bytes
